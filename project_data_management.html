Introduction to version control using git¶

How to keep track of all the things that happen to projects, including code and data? Spoiler: the classic _version1, _version2, etc. approach is not the best option. Thus, within this lecture/practice mix version control via git will be introduced and explored. This will also include aspects of simultaneous collaborative working.
